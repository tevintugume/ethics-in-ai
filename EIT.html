Fair Predictive Measures for Academy Youth in Sports

Problem Statement

In a country like Uganda, where sports in general has not become a key source of employment yet has created a gap that requires scouting and talent sourcing technology. The current methods used so far revolve around pen and paper as well as observation of players without having to consider their collective talent over a long period of time. The current academies as well have also become expensive for the youth requiring a daily payment of between 1 to 5 euros per session which is not affordable for many of the youth. This then creates a difference between children that are from well off families and poor families leaving out so many from the opportunity of becoming potential sports superstars.

As a nation with the ambition of having players in countries all over Europe and the world, this has become hard as the sports sector is considered for the poor and the educationally challenged thus leaving it in shambles yet results are required. In talent scouting and youth football development, identifying players with the highest potential is both a business and social imperative. However, recent reviews have raised concerns that the current player evaluation pipeline might be unintentionally biased, especially across gender and age especially in a 3rd world country where women are still marginalised as home managers and providers of their men and families.

The ambition of the project is to create an unbiased and fair machine learning program that will widen the net for any potential players in the country and have them participate highly on the national and international stage with in the next 5 to 10 years The goal is to design a fair and interpretable machine learning model that ranks youth football players based on their potential as well as figure out their strengths, weakneses and points of improvement. This model must not only be predictive but also ensure that it does not systematically disadvantage any demographic group. By ensuring fairness, the program will:

Increase public trust in its scouting processes.

Attract more diverse applicants.

Make better long-term investments in talent.

Reduce on the potential players that are falling through the cracks

Make the scouting process more fair and affordable for players

Documentation and clear follow up the progress of the potential players

Data is collected via a structured form filled in CustomTkinter, stored in an Excel sheet. Each record includes player demographics and performance indicators. This is framed as a regression task to predict a "prodigy score" followed by ranking players for selection.

class SoccerPlayerForm(ctk.CTk):

    def __init__(self):
        super().__init__()

        self.title("Youth Soccer Player Registration")
        self.geometry("450x600")
        self.resizable(False, False)

        # a dictionary is initialised to handle the entries from the user of the program
        self.entries = {}

        # Labels and fields are added to the code. they can be adjusted depending on the requirement of the academy or scouts
        self.fields = [
            ("Name", "text"),
            ("Age", "int"),
            ("Gender", "text"),
            ("Height (cm)", "int"),
            ("Speed (0-100 scale)", "int"),
            ("Technique (0-100 scale)", "int"),
            ("Stamina (0-100 scale)", "int"),
            ("Mental Strength (0-100 scale)", "int"),
            ("Education Grade (0-100 scale)", "int"),
            ("Sports IQ (0-100 scale)", "int"),
            ("Attitude Towards Authority (0-100 scale)", "int"),
            ("Training Drive (0-100 scale)", "int"),
        ]

        self.create_widgets()

    def create_widgets(self):
        ctk.CTkLabel(self, text="Youth Soccer Player Registration Form", font=("Times new roman", 20, "bold")).pack(pady=15)

        # Labels and entry points are generated 
        for label, field_type in self.fields:
            frame = ctk.CTkFrame(self)
            frame.pack(padx=20, pady=5, fill="x")

            ctk.CTkLabel(frame, text=label, width=180, anchor="w").pack(side="left", padx=(10, 0))
            entry = ctk.CTkEntry(frame)
            entry.pack(side="right", expand=True, fill="x", padx=(0,10))

            self.entries[label] = (entry, field_type)

        # Submit button is created to push the information to the excel sheet
        submit_btn = ctk.CTkButton(self, text="Submit", command=self.submit_data)
        submit_btn.pack(pady=20)

    def submit_data(self):
        data = {}

        # Data validation and collection is done. this section allows for the data to be taken in as well as prevent the addition of illegitimate information
        for label, (entry, field_type) in self.entries.items():
            value = entry.get().strip()

            if value == "":
                messagebox.showerror("Input Error", f"Please enter {label}.")
                return

            if field_type == "int":
                if not value.isdigit():
                    messagebox.showerror("Input Error", f"{label} must be a number.")
                    return
                value = int(value)

                # This validates the fields with a range scake of 0 to 100, it also prevents addition of figures above 100 and negative values as well 
                if "scale" in label.lower() or "grade" in label.lower():
                    if not (0 <= value <= 100):
                        messagebox.showerror("Input Error", f"{label} must be between 0 and 100.")
                        return

                if label == "Age" or label == "Height (cm)":
                    if value <= 0:
                        messagebox.showerror("Input Error", f"{label} must be a positive number.")
                        return

            data[label] = value

        # Appending to the Excel file is done without having to create an entirely new excel sheet or erasing the prior informaion within 
        self.save_to_excel(data)

        messagebox.showinfo("Success", "Player information saved successfully!")

        # Clear entries is done to prevent addition of prior information to the newly added youth candidate 
        for entry, _ in self.entries.values():
            entry.delete(0, "end")

    def save_to_excel(self, data):
        # if the file doesnt exist, a new one altogether 
        if os.path.exists(EXCEL_FILE):
            df = pd.read_excel(EXCEL_FILE)
            new_df = pd.DataFrame([data])
            df = pd.concat([df, new_df], ignore_index=True)
        else:
            df = pd.DataFrame([data])

        df.to_excel(EXCEL_FILE, index=False)

Ethical Concerns

Football has historically been male-dominated, and young female, short and small players are slipping through the cracks of many scouts and academies who would have been world class in the long run. Age bias also arises when coaches prioritize older players in a group for physical maturity. On top of that, there is concern of players having to sacrifice their educational prospects for a time in the sports life, which becomes problem when the sports dream comes to an end they are hardly qualified to do anything afterwards. Training a model without fairness checks might perpetuate these biases.

Our exploratory model showed that when using raw feature-weighted scores, top ranks were disproportionately skewed toward a specific gender and age group. This is ethically problematic:

Equal opportunity is core to fair sport.

Bias may lead to missed potential in underrepresented groups.

It violates principles of transparency and inclusion.

Fair machine learning mitigations are thus essential in this use case.

Approaches to Fairness

The following fairness-aware machine learning methods were considered:

  1. Pre-processing: Adjusting data before modeling using techniques like re-weighing or relabeling. These are simple but can distort data realism.

  2. In-processing: Applying fairness constraints during training (e.g., Fairlearn's GridSearch using Demographic Parity or Equalized Odds).

  3. Post-processing: Altering outputs to balance group outcomes. This may affect interpretability.

In-processing via Fairlearn's GridSearch with Demographic Parity was chosen, as:

It balances performance and fairness.

It does not distort input data.

It integrates well with our RandomForestRegressor base model.

Applying the Fair Model

The dataset was prepared by combining weighted features like speed, technique, stamina, and training drive to generate a raw prodigy score. We then:

Defined sensitive features: Gender, and intersectionally with Age_Group.

Split data for training/testing.

Binarized labels using a top-25% threshold.

Applied Fairlearn's GridSearch to learn fair predictions.

Normalized outputs and ranked players.

If insufficient sensitive group diversity exists (e.g., only one gender), the system defaults to raw scores and shows a warning.

Fairness metrics were evaluated:

Selection rate by group using MetricFrame.

Displayed bias summaries live in the UI.

The UI allows real-time weight adjustment and visual fairness inspection.

class FairRankingApp(ctk.CTk):
    def __init__(self):
        super().__init__()
        self.title("Fair Soccer Player Ranking with Fairlearn Model")
        self.geometry("950x750")

        try:
            self.df = pd.read_excel(EXCEL_FILE)
        except FileNotFoundError:
            messagebox.showerror("File Error", f"Could not find {EXCEL_FILE}. Please place the file in this folder.")
            self.destroy()
            return

        self.weights = {
            "Speed (0-100 scale)": 0.20,
            "Technique (0-100 scale)": 0.25,
            "Height (cm)": 0,
            "Stamina (0-100 scale)": 0.15,
            "Mental Strength (0-100 scale)": 0.15,
            "Education Grade (0-100 scale)": 0.05,
            "Sports IQ (0-100 scale)": 0.10,
            "Attitude Towards Authority (0-100 scale)": 0.05,
            "Training Drive (0-100 scale)": 0.05,
        }

        self.feature_cols = list(self.weights.keys())

        self.create_widgets()
        self.prepare_data()
        self.update_model_and_ranking()

def create_widgets(self):
        header = ctk.CTkLabel(self, text="Adjust Weights & See Fair Player Ranking", font=("Arial", 18, "bold"))
        header.pack(pady=10)

        sliders_frame = ctk.CTkFrame(self)
        sliders_frame.pack(pady=10, fill="x", padx=20)

        self.sliders = {}
        for i, (key, val) in enumerate(self.weights.items()):
            label = ctk.CTkLabel(sliders_frame, text=f"{key} Weight:", width=250, anchor="w")
            label.grid(row=i, column=0, padx=10, pady=5, sticky="w")
            slider = ctk.CTkSlider(sliders_frame, from_=0, to=1, number_of_steps=100, command=self.slider_changed)
            slider.set(val)
            slider.grid(row=i, column=1, padx=10, pady=5, sticky="ew")
            self.sliders[key] = slider
        sliders_frame.grid_columnconfigure(1, weight=1)

        self.train_button = ctk.CTkButton(self, text="Train Fair Model & Update Ranking", command=self.update_model_and_ranking)
        self.train_button.pack(pady=10)

        self.textbox = ctk.CTkTextbox(self, width=900, height=400)
        self.textbox.pack(pady=10, padx=20)

        self.bias_label = ctk.CTkLabel(self, text="", font=("Arial", 12), justify="left")
        self.bias_label.pack(pady=10)

    def slider_changed(self, value):
        total = sum(slider.get() for slider in self.sliders.values())
        if total == 0:
            return
        for key, slider in self.sliders.items():
            self.weights[key] = slider.get() / total

def prepare_data(self):
        df = self.df.copy()

        bins = [0, 12, 15, 18, 100]
        labels = ["<=12", "13-15", "16-18", "19+"]
        df["Age_Group"] = pd.cut(df["Age"], bins=bins, labels=labels, right=False)

        def compute_raw_score(row):
            s = 0
            for feat, w in self.weights.items():
                s += row[feat] * w
            return s

        df["Raw_Potential_Score"] = df.apply(compute_raw_score, axis=1)
        self.df = df

def update_model_and_ranking(self):
        self.prepare_data()

        df = self.df.copy()

        X = df[self.feature_cols]
        y = df["Raw_Potential_Score"]
        A = df["Gender"]

        X_train, X_test, y_train, y_test, A_train, A_test = train_test_split(
            X, y, A, test_size=0.3, random_state=42)

        threshold = np.percentile(y_train, 75)
        y_train_bin = (y_train > threshold).astype(int)
        y_test_bin = (y_test > threshold).astype(int)

        base_estimator = RandomForestRegressor(random_state=42)
        mitigator = GridSearch(
            estimator=base_estimator,
            constraints=DemographicParity(),
            grid_size=20,
        )
        mitigator.fit(X_train, y_train_bin, sensitive_features=A_train)

        fair_preds = mitigator.predict(X)

        min_p = fair_preds.min()
        max_p = fair_preds.max()
        prodigy_chance = 100 * (fair_preds - min_p) / (max_p - min_p)
        df["Prodigy_Chance_"] = prodigy_chance

        df_sorted = df.sort_values("Prodigy_Chance_", ascending=False)

        lines = [f"{'Rank':<5} {'Name':<20} {'Gender':<8} {'Age':<4} {'Prodigy Chance (%)':>18}"]
        lines.append("-" * 65)
        for i, row in enumerate(df_sorted.itertuples(), 1):
            lines.append(f"{i:<5} {row.Name[:18]:<20} {row.Gender:<8} {row.Age:<4} {row.Prodigy_Chance_:>17.2f}%")

        self.textbox.delete("0.0", "end")
        self.textbox.insert("0.0", "\n".join(lines))

        mf = MetricFrame(
            metrics={"selection_rate": selection_rate},
            y_true=(y > threshold).astype(int),
            y_pred=(fair_preds > threshold),
            sensitive_features=A,
        )
        bias_text = "Fairness Metric (Selection Rate) by Gender:\n"
        for g, val in mf.by_group.items():
            if hasattr(val, 'iloc'):
                val = val.iloc[0]
            bias_text += f"  {g}: {val:.3f}\n"

        mf_intersection = MetricFrame(
            metrics={"selection_rate": selection_rate},
            y_true=(y > threshold).astype(int),
            y_pred=(fair_preds > threshold),
            sensitive_features=pd.DataFrame({"Gender": df["Gender"], "Age_Group": df["Age_Group"]}),
        )
        bias_text += "\nSelection Rate by Gender + Age Group:\n"
        for key, val in mf_intersection.by_group.items():
            if hasattr(val, 'iloc'):
                val = val.iloc[0]
            if isinstance(key, tuple):
                key_str = " & ".join(str(k) for k in key)
            else:
                key_str = str(key)
            bias_text += f"  {key_str}: {val:.3f}\n"

        self.bias_label.configure(text=bias_text)
